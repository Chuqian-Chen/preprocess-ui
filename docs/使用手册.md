## 0. 准备与启动

**环境要求**：Windows / Linux / macOS，Python 3.10+，已安装 `requirements.txt` 依赖。

**启动**：
- Windows：双击 `start.bat`，浏览器自动打开 http://127.0.0.1:8877
- 手动：`python -m uvicorn server.app:app --host 127.0.0.1 --port 8877 --app-dir .`

**停止**：在服务终端按 `Ctrl+C`，或关闭窗口。

**首次先配 AI**（可选但推荐）：右上角「配置 AI Key」→ 选平台 → 填 Key → 测试连接显示成功即可。没有 AI 也能用，只是 ②③ 退化为规则模式。

## ⓪ 数据源 — 创建项目

![数据源](images/00-source.png)

两种导入方式，二选一：

**A. 服务器文件夹路径**
1. 在「文件夹路径」框填入数据所在目录的**绝对路径**（如 `E:\data\my_dataset`）。
2. 填项目名（留空则用文件夹名）。
3. 点「从路径创建」。系统会把该目录下所有 `.csv` 复制进项目的 `raw/`。

**B. 上传 CSV / ZIP**
1. 点上传区选择文件（可多选 CSV，或一个含 CSV 的 ZIP）。
2. 填项目名，点「上传创建」。

> 创建后，项目数据完全独立存放在 `workspaces/{项目id}/`，互不干扰。左侧项目列表可随时切换；切换会自动重载该项目画像。

**注意**：路径含中文时，请用界面操作（浏览器会正确编码）；用命令行/脚本传中文路径可能因编码出错。

## ① Raw 质控 — 看清每一列

![Raw 质控](images/01-raw-qc.png)

进入后点 **「扫描并生成画像」**，引擎逐列体检。三栏布局：

- **左：数据表列表** — 每张表的字段数；超大表标「采样 N 万行」。
- **中：字段画像表** — 每行一列，含 `字段 / 类型 / 缺失% / 去重 / 内容摘要`。
- **右：值域预览** — 点中间任一行，这里展示该列的取值分布（频次条形图 / 数值统计 / 日期范围）。

### 改类型
中间「类型」列是下拉，可手动改（如把被误判为日期的 ID 改成「ID型」）。**改后值域即时重算**，右侧预览同步更新。

类型说明：
| 类型 | 自动判定（≈） | 值域展示 |
|---|---|---|
| 空列 | 全部缺失 | — |
| 数值型 | 90%+ 可转数字 | min/max/均值/中位 |
| 日期型 | 90%+ 可转日期 | 最早 / 最晚 |
| 分类/枚举型 | 取值很少（≤50 类或唯一率<1%） | Top 频次条形图 |
| ID型 | **需手动指定** | Top 样例频次 |
| 文本型 | 其余 | 去重数；≤200 类时给频次 |

### 保存与下载
- **「保存类型配置」**：把你所有改过的类型持久化（写入项目 `config/schema_overrides.json`）。
- **「下载质控 CSV」**：导出含已保存类型修改的字段画像总表（`表名/字段/类型/缺失数/缺失%/去重数/是否采样/内容摘要`），可在 Excel 里离线查阅。

> ⚠️ 采样表的缺失%、去重、频次是**前 20 万行的样本估计**，量级可信、绝对值有偏差。需要精确请关采样（见 README）后重扫。

## ② 逻辑关联 — 表怎么连起来

![逻辑关联](images/02-join.png)

两个按钮：

**「统计探查」**（无需 AI）
- 计算各表 ID 类列两两之间的**列值交集匹配率**，给出候选关联列对（左表.列 ↔ 右表.列 + 匹配率）。

**「AI 识别关联」**（需 AI）
- 把字段画像（含关键 ID 列的完整 top 值域）+ 探查结果 + 数据样例喂给 AI，生成：
  - **关联规则表**（路径 / 步骤 / 左右表字段 / 样例值 / 匹配率 / 备注）
  - **Mermaid 关联路径图**
  - 关键 ID、常见误区提示

### 值域速查（改规则的利器）
页面底部「值域速查」卡片：
- 顶部**搜索框**可跨表搜字段名/表名，下拉选表→选字段，下方即时画出该字段的分布。
- 规则表里的**左字段/右字段是可点链接**，点一下自动跳到值域速查并展示该字段——边看真实取值边判断这条 join 对不对。

### 编辑与保存
- 展开「高级：编辑 JSON」可手改规则。
- 点「保存」写入项目 `config/join_rules.json`（同时导出 `关联规则与示例.csv` 格式）。

## ③ AI 预处理 — 逐列清洗

这是核心清洗步。**raw 永不被修改**，所有结果写到 `output/`。

![AI 预处理](images/03-preprocess.png)

布局：左=表列表，中=字段+修改意见，右=审阅状态（含可折叠值域）。

**操作流程**：
1. （可选）顶部「本表补充要求」框填整体诉求，如「保留关键 ID 列、删除冗余 ID、统一日期格式」——会传给 AI。
2. 点 **「AI 生成本表意见」**：AI 为每列生成一句修改建议草稿（如「保留，作为关联键」「统一日期格式」「删除」），来源标 `AI`。
3. **审阅编辑**：中间每列的「AI 修改建议」是文本框，你可改可清空。点左侧/中间某行，右侧「审阅状态」展示该列值域（可点标题折叠），方便对照。
4. 点 **「保存审阅结果」**：把你的编辑落盘（标记为已审阅）。
5. 点 **「确认应用本表」**：把意见**编译成 JSON 操作**并执行，生成/更新 `output/{表}_clean.csv`。

> 二次应用时，以上次的 output 为输入，可反复迭代清洗。

**意见怎么写**：自然语言即可（AI 会编译）。无 AI 时请用可被规则识别的句式：`改名为 XXX` / `删除` / `合并为 XXX` / `去空格`，否则会提示无法解析。

**支持的底层操作**（AI 最终只会产出这些，安全可控）：
`rename_column / drop_columns / merge_columns / fillna / map_values / strip_whitespace / dedupe / filter_rows / select_columns`。

## ⑤ 数据分析 — 可视化

![数据分析](images/05-analysis.png)

三个视图（切换 scope）：

- **Raw 分析**：选表后看该表的①类型分布、②各列缺失率、③选某列看分布（数值=直方图，分类/ID=频次条）。
- **处理后分析**：同上，但读 `output/` 里清洗后的表（需先在 ③ 应用过）。
- **对比**：同一张表 raw vs 处理后——行/列数变化、删除/新增列、各列缺失率并排、选列看分布前后对比。

> 图表由本地 Chart.js 渲染，离线可用。分析同样基于采样读取，速度快。

## 常见问题（FAQ）

**Q：切了新项目，Raw 质控还显示旧表？**
A：已修复。切项目会即时清空并显示「画像计算中…」。超大表首次画像可能要等（已采样加速）；算完缓存进 `profile.json`，之后秒开。

**Q：AI 报 401 Authorization Required？**
A：多为 key 与平台不匹配（拿 A 家 key 敲 B 家 url）。本工具已做「同源解析」：优先级 `PREPROCESS_AI_*` 环境变量 > 本地文件 > `OPENAI_API_KEY`，三者整组取用。检查右上角配置或删掉 `config/ai_settings.local.json` 重配。

**Q：数据会上传到云端吗？**
A：不会。数据全程在本机 `workspaces/`。仅当你使用 AI 功能时，**字段画像摘要 + 少量样例**会发给你配置的 AI 服务用于生成建议；原始数据本身不整表上传。涉敏感数据请谨慎选择 AI 平台。

**Q：换电脑/离线机怎么搬？**
A：拷贝整个 `preprocess_ui` 文件夹（含 `workspaces/` 才带数据），目标机装好 Python 依赖即可跑。前端图表库已本地化，断网无碍。

**Q：误改了类型/意见想重来？**
A：类型改回即可重算；清洗结果删掉 `output/` 对应文件即回到读 raw 的初始态。raw 始终未变。
